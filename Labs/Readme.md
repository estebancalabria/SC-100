# Laboratorios SC-100

Este repositorio recopila los laboratorios (case studies) del examen **SC-100: Microsoft Cybersecurity Architect**.

Los labs están enfocados en escenarios reales donde se deben diseñar soluciones de seguridad a nivel arquitectónico, cubriendo identidad, datos, aplicaciones, infraestructura y cumplimiento.

---

## 🧪 Lista de Laboratorios

### 1. Case Study: Introducción
**Objetivo:** Entender el contexto del negocio, requisitos y restricciones del escenario.  
**Resumen:** Se presenta la organización, su estado actual y desafíos de seguridad. Sirve como base para todos los demás diseños.

---

### 2. Design a Zero Trust Solution
**Objetivo:** Diseñar una arquitectura basada en el modelo Zero Trust.  
**Resumen:** Se aplican principios como "verify explicitly", "least privilege" y "assume breach", integrando identidad, dispositivos, aplicaciones y datos.

---

### 3. Architecture Best Practices
**Objetivo:** Aplicar buenas prácticas de arquitectura de seguridad.  
**Resumen:** Se evalúan patrones seguros de diseño en Azure y Microsoft 365, alineados con estándares modernos.

---

### 4. Design Solutions aligned with Cloud Adoption Framework (CAF)
**Objetivo:** Alinear la seguridad con el Cloud Adoption Framework.  
**Resumen:** Se integran prácticas de gobernanza, identidad y seguridad dentro del ciclo de adopción cloud.

---

### 5. Design Solutions with MCRA and MCSB
**Objetivo:** Aplicar Microsoft Cybersecurity Reference Architecture (MCRA) y Microsoft Cloud Security Benchmark (MCSB).  
**Resumen:** Se diseñan soluciones siguiendo estándares de seguridad de Microsoft para workloads cloud.

---

### 6. Design a Resiliency Strategy for Ransomware
**Objetivo:** Diseñar una estrategia contra ransomware.  
**Resumen:** Se abordan backup, recuperación, segmentación y detección para minimizar impacto de ataques.

---

### 7. Evaluate Regulatory Compliance
**Objetivo:** Evaluar cumplimiento normativo.  
**Resumen:** Se utilizan herramientas como Microsoft Purview para cumplir regulaciones (GDPR, etc.).

---

### 8. Design an Identity Security Solution
**Objetivo:** Diseñar seguridad de identidad.  
**Resumen:** Incluye autenticación, MFA, Conditional Access y protección de identidades en Microsoft Entra ID.

---

### 9. Design Solutions for Securing Privileged Access
**Objetivo:** Proteger accesos privilegiados.  
**Resumen:** Uso de PIM (Privileged Identity Management), Just-In-Time y controles de acceso elevados.

---

### 10. Design Solutions for Security Operations
**Objetivo:** Diseñar operaciones de seguridad (SOC).  
**Resumen:** Integración con Microsoft Sentinel y Defender para detección, respuesta y monitoreo.

---

### 11. Design Solutions for Securing Microsoft 365
**Objetivo:** Proteger el entorno Microsoft 365.  
**Resumen:** Configuración de seguridad para Exchange, SharePoint, Teams y protección contra amenazas.

---

### 12. Design Solutions for Securing Applications
**Objetivo:** Asegurar aplicaciones.  
**Resumen:** Incluye identidad de apps, secretos, API security y protección en runtime.

---

### 13. Design Solutions for Securing Data
**Objetivo:** Proteger datos sensibles.  
**Resumen:** Uso de clasificación, etiquetado, DLP y cifrado.

---

### 14. Specify Requirements for SaaS, PaaS and IaaS Security
**Objetivo:** Definir requisitos de seguridad para distintos modelos cloud.  
**Resumen:** Diferencias de responsabilidad compartida y controles necesarios según el tipo de servicio.

---

### 15. Design Security Posture Management & Threat Intelligence
**Objetivo:** Gestionar postura de seguridad y amenazas.  
**Resumen:** Uso de Defender for Cloud y herramientas de inteligencia de amenazas.

---

### 16. Design Solutions for Securing Endpoints
**Objetivo:** Proteger dispositivos y endpoints.  
**Resumen:** Estrategias de acceso remoto, hardening y protección con Defender for Endpoint.

---

## 🧠 Conclusión

Los laboratorios de SC-100 no son hands-on tradicionales, sino **escenarios de diseño arquitectónico**, donde se evalúa:

- Pensamiento estratégico
- Conocimiento de herramientas Microsoft
- Capacidad de integrar múltiples dominios de seguridad

Son clave para preparar el examen, ya que replican el formato de **case studies reales**.
